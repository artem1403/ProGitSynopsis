Глава 3. Git на сервере

Объяснение протоколов не буду фиксировать, как и процесс установки серверной части Git.

Объединение локального репозитория и удаленного
	$ git remote [add gitlab https://server/namespace/project.git] - для действующего локального репозитория создаст удаленный репозиторий.
	в пространстве namespace с названием project.
	$ git clone https://server/namespace/project.git - получить с сервера рабочую копию проекта project.

Участие в проекте (Contributing to a Project)
	$ git diff [--check] - показывает строки, с пробелами в конце, которые нужно убрать. Сделайте это сразу.
	$ git add [--patch] - индексирование изменений в файлах ЧАСТЯМИ. Git показывает каждый блок изменений и спрашивает, добавить его в индекс или нет.

	Сообщения коммитов желательно делать менее 50 символов, кратко и по делу.
	Коммитить изменения логическими частями, часто, а не собирать все доработки в 1 коммит.

	$ git log --no-merges [myBranchName]..origin/master - посмотреть коммиты, которые придется мерджить при слиянии ветки myBranchName и master.

	$ git request-pull - создать запрос на слияние, но нужно понять синтаксис, как это работает
